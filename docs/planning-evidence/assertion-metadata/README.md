# Assertion Metadata Evidence

This hierarchy mirrors compact assertion organization across planning stages.
Each entry owns the assertion's unique name/ID, kind, provenance, temporal
scope, extraction coverage, current stage, and assertion-list link.

Metadata does not validate an assertion or change its lifecycle stage by
itself. The [canonical planning protocol](../../planning/README.md) governs all
transitions.
