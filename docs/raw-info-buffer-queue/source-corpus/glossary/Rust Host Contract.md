A Rust Host Contract is the Rust-defined contract surface that tells the [[Capability Framework]] how a capability-side
type, trait, callback, extension slot, data surface, or native binding is validated, projected, materialized, scheduled,
or executed.

Rust Host Contracts are the normal bridge between typed Rhai declaration material and Rust-native implementation
surfaces.
They may be generated by macros, registered by built-in code, supplied by a [[Capability Kernel]], or adapted around
external Rust crates.

Responsibilities can include:

- validating [[Rhai Asset]] declaration data
- defining or checking signature metadata
- projecting script-safe or callback-safe `ctx` surfaces
- binding declarations to [[Scriptable Rust Surface]] entries
- materializing [[Capability Instance]]s
- scheduling or mediating [[Capability Callback]] invocation
- routing native execution into a [[Capability Kernel]]
- participating in host-side reconcile/commit/apply policy

Boundary:
Rust Host Contract is not the same thing as Capability Kernel.
The host contract defines the shape, validation, projection, and binding rules.
The kernel is native implementation backing for module-scoped execution where one is needed.
