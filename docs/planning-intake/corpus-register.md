# Planning Source Corpus Register

This register lists every source artifact currently admitted to the
planning-intake process and its atomization state. The corpus remains open-ended;
more sources may be added later.

The canonical lifecycle rules live in the
[planning protocol](../planning/README.md). This file records source state only.
Source state never implies truth, freshness, consistency, or authority.

## States

- `unprocessed`: the source has not been completely atomized.
- `fidelity-audit-required`: candidates exist, but their extraction must be
  checked against the current source-verbatim rule.
- `atomized`: every substantive passage has been represented by atomized
  candidates or explicitly accounted for as non-assertive structure,
  within-source duplication, or source-document metadata.

Even an `atomized` source remains entirely unvetted. Only exhaustive candidate
review can place an assertion in active planning.
