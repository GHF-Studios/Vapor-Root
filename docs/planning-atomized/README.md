# Atomized Planning

This stage contains atomic candidates that have not been vetted. Candidates may
be wrong, stale, contradictory, redundant, incomplete, or poorly named.

Presence here means only that a candidate must eventually be considered under
the [canonical planning protocol](../planning/README.md).

Each candidate occupies one compact linked line. Its name and lifecycle
metadata live under the matching hierarchy in
[`planning-evidence/assertion-metadata/`](../planning-evidence/assertion-metadata/).
