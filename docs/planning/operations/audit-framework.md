# Operation — Audit Framework

## Invoke

```text
Audit framework.
```

## Behavior

Inspect the bootstrap, canonical protocol, intake register, atomized planning,
active planning, superseded planning, operations, extraction records, review
records, navigation, and current Git state for contradictory lifecycle rules,
obsolete terminology, missing source entries, broken links, missing evidence,
invalid gates, ambiguous interaction behavior, and stale staged/working copies.

## Boundaries

This is an operational review, not semantic vetting. Do not alter assertion
meaning, atomize sources, promote candidates, or validate project claims.
Repair files only when the user explicitly requests repair; otherwise report
findings first.

## Pivot conditions

Pause when a repair requires product meaning, owner policy, destructive cleanup,
or changing the foreground workstream.

## Result

Return findings ordered by impact, identify any blocker, and state whether the
framework can safely perform its next requested operation.
